About the app

TANDEM is an environment that generates quantitative image and text data from files submitted by the user. This output is intended to be used as source material for data visualization, quantitative analysis, and distant reading of multimodal print objects.

TANDEM compiles open source technologies, including Tesseract OCR, image feature extraction, and natural light language processing packages to generate data output as CSV. The final output also contains a downloadable folder of the OCR text, word lists, and more.

To explore the functionality of TANDEM, our team employed a test corpus of Mother Goose picture books in the public domain. To view the source and results from the initial corpus, see: Sample Data.

TANDEM intends to provide scholars a singular source for quantitative insight into picture books, comics, illuminated manuscripts, and other sources that might contain both image and overlaid text. That said, this application was built so that anyone can be free to use and explore this tool.

Frequently Asked Questions

What does TANDEM solve?

TANDEM was built to make the process of acquiring multimodal data from text and image content easier.

How was TANDEM built?

The TANDEM team used Python to wrap Tesseract OCR, NLTK, and OpenCV into a single, streamlined Django application. Previously such an multi-pronged analysis would have required each program to be run individually through the command line.

What are the components of TANDEM?

  • NLTK

    • NLTK stands for the Natural Language Toolkit. It is a free, open source, community-driven project which contains a suite of libraries and programs for symbolic and statistical natural language processing (NLP) for the Python programming language. Such libraries include: Lexical analysis: Word and text tokenizer, n-gram and collocations, part-of-speech tagging, and named-entity recognition. Learn more about NLTK.

  • Open CV

    • OpenCV stands for Open Source Computer Vision and focuses on real-time image processing. The OpenCV library has more than 2500 optimized algorithms, which can be used to detect and recognize faces, identify objects, classify human actions in videos, track camera movements, track moving objects, find similar images from an image database, follow eye movements, recognize scenery and establish markers to overlay it with augmented reality, and more. Learn more about Open CV.

  • Tesseract OCR

    • Tesseract OCR is an optical character recognition engine, sponsored by Google since 2006. Tesseract can read a wide variety of image formats and convert them to text in over 60 languages. Read more about Tesseract OCR.

I've downloaded my data output. What does each column in the CSV represent?

Column Header Data Description
File File name from your original upload
Total Word Count Total number of words across a particular image
Total Characters Total number of characters including letters and punctuation across a particular image
Average Word Length Average number of characters in the words across a particular image
Unique Word Count Number of different words used across a particular image
Count without Stopwords Total number of words after stopwords, or words that occur with high-frequency and have less apparent significance such as “the” and “also”
Image Size (px) Total number of pixels of a given image
Image Width (px) Side to side length of the image in pixels
Image Height (px) Top to bottom length of the image in pixels
Image Color Refers to the color model of the image*
Mean R Average brightness intensity value of the color red in the image
Mean G Average brightness intensity value of the color green in the image
Mean B Average brightness intensity value of the color blue in the image
Mean Color Channel Will often indicate a value of zero because all images across a volume will generally be the same generating a mean of 0
Std Dev Mean R
Std Dev Mean G
Std Dev Mean B
Used to calculate how dispersed the color values are across all the pixels of the image**

*A value of “3” indicates that the image has 3 color channels. If it is an RGB image those values are a red, green, and blue channel. Most digital images are formatted in RGB. Red, green, and blue values mix together to generate the rest of the color spectrum on digital devices.

**A value closer to zero will illustrate that the image is generally made up of the same colors. Higher standard deviation values in these columns will indicate that there are more colors used across each channel and therefore a more complex set of pixels.
